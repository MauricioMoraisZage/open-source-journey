## Process

For each project, I will:

1. Read the README and contribution guide
2. Understand the repository structure
3. Prepare the local environment
4. Run the existing tests
5. Review beginner-friendly issues
6. Select a small and available issue
7. Create a dedicated branch
8. Implement and test the change
9. Open a clear pull request
10. Respond to maintainer feedback

## First Contribution Strategy

My first contribution should be small, focused and properly tested.

Possible starting points:

- Documentation improvements
- Test improvements
- Typing fixes
- Small bug fixes
