## Contribution Process

For each selected project, I will:

1. Read the README and contribution guide
2. Understand the repository structure
3. Prepare the local development environment
4. Install project dependencies
5. Run the existing checks and tests
6. Review beginner-friendly issues
7. Select a small and focused issue
8. Create a dedicated branch
9. Implement the change
10. Run the required checks locally
11. Open a clear pull request
12. Respond to maintainer feedback
13. Document the contribution and lessons learned

## First Contribution Strategy

My first contributions should be small, focused and easy to review.

Good starting points include:

* Documentation improvements
* Test improvements
* Typing fixes
* Small bug fixes
* Issue reproduction
* Examples improvement
